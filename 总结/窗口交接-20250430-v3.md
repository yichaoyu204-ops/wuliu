## 3. 首次错误与纠正

### 错误1："所有运单"功能给所有角色可见
- **第一次怎么理解**：首页功能入口所有人都能看到
- **为什么不符合意图**：漏掉了"只有管理员界面才有全部运单"这句话
- **用户怎么纠正**：直接说"在管理员界面下是限制，只有管理员界面改"
- **正确做法**：用 wx:if 控制，只在 isAdmin 时显示第五个 tab

### 错误2：计费逻辑未先转吨
- **第一次怎么理解**：直接用公斤（3411.95）算 ratio = volume / weight
- **为什么错**：漏掉了"除1000转吨"这个关键步骤
- **用户怎么纠正**：明确说"先把重量变为吨，因为测出来是公斤"
- **正确做法**：weightTons = weightKg / 1000，ratio = volume / weightTons
- **为什么纠正后仍不对**：第一遍只改了公式，没改显示文案（还是显示"公斤"而不是"吨"）

### 错误3：云函数部署后仍报"网络错误"
- **第一次怎么理解**：部署了就生效，可能是代码逻辑问题
- **为什么错**：notification.js 引用了不存在的 ./aiParser 模块
- **排查过程**：让用户在前端 catch 块打印错误 → 发现 Cannot find module './aiParser'
- **纠正**：把 truncatePushSummary 内联到 notification.js 中，删除 require('./aiParser')

### 错误4：权限不足（白名单为空）
- **第一次怎么理解**：空数组应该放通所有用户（部署文档写了）
- **为什么错**：代码中没有实现"空数组放通"逻辑
- **用户怎么纠正**："我妈妈是仓管员，却说我权限不足"
- **纠正**：在 isWarehouseOrAdmin 和 isAdmin 中添加 whitelistEmpty 判断

### 错误5：把"仓库员"文案漏改为"仓管员"
- **第一次怎么理解**：只改了前端显示，忘了改云函数中的错误提示
- **为什么错**：grep 搜索不完整
- **纠正**：全局 grep + sed 批量替换所有"仓库员"为"仓管员"

## 4. 根因分析

| 类型 | 具体问题 | 根因 |
|------|---------|------|
| 需求理解 | "所有运单"给所有人看 | 把"管理员限定"理解成了"首页通用" |
| 上下文遗漏 | 漏掉 CLAUDE.md 切换窗口确认指令 | 没先确认用户是否有未完成内容 |
| 执行偏差 | 云函数修改后没提醒部署 | 默认用户知道要部署 |
| 过度简化 | 物流节点和 OA 流程混为一谈 | 没意识到这是两个独立的进度系统 |
| 格式交付 | 11项总结写得太短 | 低估了用户对详细程度的要求 |
| 沟通确认 | 计费逻辑只改了一遍 | 没和用户逐字确认公式每一步 |

## 5. 用户真正想要的效果

仓管员、管理员、业务员各自看到自己该看的、做自己该做的，物流状态实时同步给所有人，但权限严格分离。用户体验要像"傻瓜式操作"，每个按钮点下去都是当前最合理的下一步。

## 6. 下次避免规则

1. **涉及多角色时，先画流程图再写代码** — 不要直接按字面理解动手
2. **云函数修改后立刻提醒部署** — 不要等用户报错
3. **提到"所有角色"时，逐一确认每个角色的差异点**
4. **计费逻辑必须和用户逐字核对公式** — 每行代码对应一句用户原话
5. **11项总结必须详细到每个错误的前因后果** — 不能偷懒
6. **全局替换前先搜索确认范围** — 避免漏改云函数中的文案
