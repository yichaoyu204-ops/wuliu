## 当前架构

- 微信小程序（原生）+ 微信云开发（云函数 + 云数据库）
- AI识别：Kimi API（moonshot-v1-8k-vision-preview）
- OA状态机：created → measured_priced → admin_confirmed → completed
- 物流节点：pickup（已提货）→ yiwu_entry（已入库）→ mainline（已出库）→ nb_entry（已完成入库）

## 三角色业务流程（最新）

### 业务员
- 建单（手动 / AI识别进仓单）
- 在「我的运单」中只读查看自己创建的运单
- 萌恒月结、仓库现结都不需要业务员二次确认

### 仓管员
- **待处理**：active = pickup 的订单
  - 实测定价（仓库现结实测后弹出收款码，确认后自动 admin_confirmed）
  - 确认入库（推进 pickup → yiwu_entry）
- **已入库**：active = yiwu_entry 的订单
  - 不强制等管理员确认价格也可以推进出库
  - 确认出库（推进 yiwu_entry → mainline）
- **已完结**：active 为 mainline/nb_entry 或 status=completed
  - 仅查看详情

### 管理员
- 流程页只显示需要他「确认价格」的订单（萌恒月结）
- 不显示「确认入库」按钮
- 仓库现结的订单不进入管理员待处理列表（仓管员实测时已自动确认）
- 全部运单 / 删除 仅管理员可见

## 计费逻辑（已确认）

1. 重量(kg) ÷ 1000 = 吨
2. 体积(立方) ÷ 吨 = ratio
3. ratio ≥ 4 → 轻货（按体积），ratio < 4 → 重货（按吨）
4. 萌恒仓库：轻货 55元/立方，重货 190元/吨
5. 北仑其他：轻货 85元/立方，重货 220元/吨

## 关键功能模块

| 模块 | 当前实现 |
|------|---------|
| AI 识别 | Kimi API + prompt 校验：业务编号必须含 MH、开航时间晚于截仓、重量/体积合理性 |
| 收款码 | 仓库现结实测后弹出占位二维码，等待用户提供真实图片 |
| 实测照片跨角色访问 | shipment-query 详情接口自动转 cloud:// 为 HTTPS 临时 URL |
| 路线编辑 | 详情页可切换路线，路线变更时若 timeline 未推进则自动重置 |
| 删除 | 物理删除运单 + 关联日志/账单/司机记录 |
| 业务编号去重 | 创建时检查重复，但已删除的不算 |
| badge 提醒 | 只统计仓管员当前真正有权限操作的订单 |
| 状态文案 | 前端映射 oaStatus，旧数据也能显示新文案 |
| 物流节点状态 | yiwu_entry/mainline 显示「待入库/已入库/待出库/已出库」，其他显示进行中/已完成 |

|------|-----------|---------------|
| pickup | 进行中 | 已完成 |
| yiwu_entry | 待入库 | 已入库 |
| mainline | 待出库 | 已出库 |
| nb_entry | 进行中 | 已完成 |

## 状态文案映射（前端覆盖）

```
created           → 已建单，待仓管员实测定价
measured_priced   → 已实测定价，管理员同步确认中
admin_confirmed   → 管理员已确认，待送达
completed         → 流程已结
```

仓库现结实测后特殊文案：**已实测定价并支付，待入库**

## 关键约束

- WXML 不支持可选链 `?.` / 空值合并 `??`
- 云存储 fileID 默认只有上传者能访问，云函数侧需转 HTTPS 临时 URL
- 云开发集合不存在时直接 .where 会抛错，需 try-catch 包裹
- shipment-create 必须 `const _ = db.command;`，否则去重查询报 `_ is not defined`
- 国内微信生态，云开发免费额度有限
