# 一超物流智能追踪系统 - 完整部署指南

## 一、文件说明

### 可安全修改的文件（不会影响历史数据）

| 文件 | 说明 | 修改影响 |
|------|------|----------|
| `后端云函数/00-全局配置/config/index.js` | 路线模板、AI提示词、费用标准、模板ID | 只影响新创建的货物 |
| `后端云函数/05-工具函数/utils/notification.js` | 通知文案 | 立即生效，不影响数据 |
| `后端云函数/05-工具函数/utils/response.js` | API响应格式 | 立即生效，不影响数据 |
| `前端小程序/` | 所有前端页面 | 立即生效，不影响数据 |

### 不可随意修改的文件（会影响现有功能）

| 文件 | 说明 | 修改风险 |
|------|------|----------|
| `后端云函数/*/index.js` | 核心业务逻辑 | 可能破坏现有功能 |
| `后端云函数/*/package.json` | 依赖配置 | 可能导致云函数部署失败 |

## 二、部署步骤

### 步骤1：准备工作

1. **注册微信小程序账号**
   - 访问 https://mp.weixin.qq.com/
   - 注册企业/个体户小程序（需要营业执照）
   - 完成微信认证

2. **开通云开发**
   - 登录小程序后台
   - 点击"云开发"按钮开通
   - 记录云环境ID（后面要用）

3. **申请Kimi API Key**
   - 访问 https://platform.moonshot.cn/
   - 注册账号
   - 创建API Key并保存

### 步骤2：配置环境

1. **修改小程序云环境ID**
   ```javascript
   // 编辑 前端小程序/app.js
   wx.cloud.init({
     env: '替换为你的云环境ID',
     traceUser: true
   });
   ```

2. **部署云函数**
   - 打开微信开发者工具
   - 导入项目（选择 前端小程序 文件夹）
   - 右键点击每个云函数 → "创建并部署：云端安装依赖"
   - 部署顺序（有依赖关系）：
     1. `config`（无依赖）
     2. `utils/*`（工具函数，被其他云函数引用）
     3. `ai-parse`、`shipment-create`、`shipment-query`、`shipment-update`、`fee-config`、`billing-manage`

3. **设置云函数环境变量**
   - 打开云开发控制台
   - 进入"云函数" → "设置"
   - 添加环境变量：
     ```
     KIMI_API_KEY=你的API密钥
     ADMIN_LIST=管理员OpenID1,管理员OpenID2
     ```
   - `ADMIN_LIST` 用于 `fee-config` 的权限校验，逗号分隔多个 OpenID

### 步骤3：创建数据库

在云开发控制台中创建以下集合：

1. **shipments** - 货物主表
2. **users** - 用户表
3. **operation_logs** - 操作日志
4. **fee_categories** - 费用类目表
5. **bills** - 账单表

创建索引：
```javascript
// shipments 集合
- managerPhone (升序) + createdAt (降序)
- shareToken (升序)
- measurement.status (升序) + createdAt (降序)   // 仓库待测量队列查询

// users 集合
- phone (升序)

// operation_logs 集合
- shipmentId (升序) + timestamp (降序)

// fee_categories 集合
- isEnabled (升序) + sortOrder (升序)   // 列表查询

// bills 集合
- clientPhone (升序) + createdAt (降序)   // 客户查我的账单
- status (升序) + createdAt (降序)        // 管理端查未付账单
```

### 步骤4：申请微信订阅消息模板

1. 登录微信公众平台
2. 进入"功能" → "订阅消息"
3. 申请以下两个模板：
   - 搜索"物流" → "物流状态更新提醒" → 复制模板ID到 `config/index.js` 的 `shipmentUpdate`
   - 搜索"报价"或"确认" → 申请报价确认通知模板 → 复制模板ID到 `config/index.js` 的 `quoteConfirmation`

### 步骤5：配置角色权限

#### 管理员 OpenID
编辑 `后端云函数/06-费用类目云函数/fee-config/index.js`：
```javascript
const ADMIN_OPENIDS = [
  'o-xxxxxxxxxxxxxxxxxxxxx',  // 你的OpenID
];
```

编辑 `后端云函数/04-节点更新云函数/shipment-update/index.js`：
```javascript
const ADMIN_OPENIDS = [
  'o-xxxxxxxxxxxxxxxxxxxxx',  // 你的OpenID
];
```

#### 仓库员 OpenID
编辑 `后端云函数/04-节点更新云函数/shipment-update/index.js`：
```javascript
const WAREHOUSE_OPENIDS = [
  'o-xxxxxxxxxxxxxxxxxxxxx',  // 仓库员OpenID
];
```

> 白名单为空数组时，`shipment-update` 的 `isWarehouseOrAdmin` 会**放通所有用户**（方便调试）。**上线前必须填写。**

### 步骤6：配置定时触发器

`billing-manage/config.json` 已包含配置：
```json
{
  "triggers": [
    {
      "name": "cleanupPaidBillsTrigger",
      "type": "timer",
      "config": "0 0 1 * * *"
    }
  ]
}
```

部署 `billing-manage` 云函数后，在云开发控制台 → 云函数 → `billing-manage` → 触发器管理中确认触发器已生效。

## 三、完整测试流程

### 测试1：建单
- 在"建单"页面上传进仓通知单或手工录入
- 验证AI是否正确提取字段
- 验证单号生成格式
- 验证分享链接是否正常

### 测试2：仓库实测
- **仓库员**打开"实测录入"页面
- 选择待测量的货物
- 填写实测数据（件数、重量、体积）
- 上传称重/量方照片（至少1张）
- 提交成功

### 测试3：费用类目配置（管理员）
- 打开"费用类目管理"
- 查看默认类目是否自动播种
- 修改一个单价（如提货费从100改到120）
- 验证价格变更留痕（去数据库查看 `priceHistory`）

### 测试4：生成报价（管理员）
- 打开"报价生成"页面
- 选择已实测的货物
- 系统自动按费用类目计算金额
- 核对单位一致性（申报 vs 实测）
- 添加其他费用（可多行）
- 保存草稿 → 发送报价
- 验证客户是否收到订阅消息通知

### 测试5：客户确认报价
- **客户**收到通知，点击进入确认页
- 查看货物信息、实测照片、费用明细
- 选择支付方式：
  - **现结**：显示"提货前扫描仓库公示码支付"
  - **月结**：显示"本月入账，次月统一结算"
- 点击确认
- 验证数据库：shipment 状态变为 confirmed，bills 集合新增账单

### 测试6：客户查看账单
- **客户**打开"我的账单"
- Tab 切换：全部 / 待付 / 已付
- 查看账单详情（单号、工厂、金额、状态）

### 测试7：管理端标记已付
- **管理员**打开"账单总览"
- 查看统计：未付总额、已付总额
- Tab 切换筛选
- 对未付账单点击"标记已付"
- 验证数据库：bill 状态变为 paid，shipment billing 同步更新

### 测试8：月结合并
- **管理员**点击"生成月结账单"
- 选择月份（如 2026-04）
- 输入客户手机号
- 确认生成
- 验证：系统合并该客户当月所有现结账单，生成一张月结总账单

### 测试9：定时清理
- 手动调用 `billing-manage` action=`cleanupPaidBills`
- 验证：已付超过90天的账单被软删除（`isDeleted: true`）

## 四、常见修改场景

### 场景1：添加新的路线模板

编辑 `后端云函数/00-全局配置/config/index.js`：

```javascript
defaultRoutes: [
  // 现有路线...
  {
    id: 'route_new_xxx',

    description: '路线描述',
    nodes: [

        code: 'step1',
        name: '节点名称',
        minDuration: 1,
```

- 修改单价、计费方式

编辑 `后端云函数/05-工具函数/utils/notification.js`。

## 六、故障排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| AI识别失败 | API Key未配置 | 检查云函数环境变量 |
| 云函数超时 | 默认超时3秒 | 修改config.json为20秒 |
| 数据库查询慢 | 缺少索引 | 按指南创建索引 |
| 通知发送失败 | 用户未订阅 | 使用"复制文案"兜底 |
| 分享链接失效 | Token错误 | 检查数据库shareToken |
| 费用类目创建失败 | 权限不足 | 检查 ADMIN_OPENIDS 和 ADMIN_LIST 环境变量 |
| 报价确认失败 | 报价状态不对 | 确保先保存草稿再发送，客户才能确认 |
| 账单查询为空 | 缺少索引 | 检查 bills 集合的 clientPhone + createdAt 索引 |
| 月结生成失败 | 无现结账单可合并 | 确保该客户有 status=unpaid/paid 且未归入月结的现结账单 |
