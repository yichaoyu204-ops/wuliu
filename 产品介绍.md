# 一超物流智能追踪系统

---

## 一、项目背景与出发点

### 1.1 行业痛点

传统物流货代行业长期面临以下问题：

| 痛点 | 具体表现 | 影响 |
|------|----------|------|
| **信息不透明** | 货物发出后，客户只能通过电话/微信询问进度 | 客户焦虑，客服压力大 |
| **效率低下** | 单据信息人工录入，易出错、耗时长 | 人力成本高，错误率高 |
| **沟通成本高** | 业务员每天花大量时间回复"货到哪了" | 无法专注核心业务 |
| **竞争同质化** | 价格战激烈，缺乏差异化服务能力 | 利润薄，客户忠诚度低 |

### 1.2 产品目标

> **用数字化手段，将传统货代的服务流程标准化、透明化，让客户像查快递一样随时掌握货物状态，从而在传统货代市场中建立差异化竞争力。**

### 3.2 核心功能详解

#### 功能1：AI智能识别（差异化亮点）

**流程：**
```
拍照/PDF上传 → AI提取字段 → 图片放大优化 → 人工校对 → 确认建单
```

**支持格式：**
- 图片：jpg、jpeg、png、gif、bmp、webp、tif、tiff、heic、heif
- 文档：PDF（从微信聊天选文件）

**价值点：**
- 3秒完成传统5分钟的录入工作
- 自动检测手写/涂改，黄色高亮提醒，避免出错
- 支持不同仓库格式的进仓通知单
- 图片短边自动放大至1024px，确保AI能看清小字
- 唛头识别优化，支持跨行内容提取

**技术实现：**
- 调用Kimi多模态大模型（moonshot-v1-8k-vision-preview）
- 提示词工程优化识别准确率
- 字段级置信度评估
- 自动校验与修正机制（validateAndFixFields）

#### 功能2：OA流程管理（简化版4状态）

```
已创建(created)
    ↓ 仓库员实测定价
已实测定价(measured_priced)
    ↓ 管理员确认
管理员已确认(admin_confirmed)
    ↓ 仓库员确认送达
流程已结(completed)
```

**流程说明：**
- 删除工厂确认环节，简化流转
- 仓管员实测后自动触发定价计算
- 管理员一键确认，无需工厂介入
- 现结/月结统一在送达环节处理

**物流节点追踪：**

| 路线 | 节点 | 适用场景 |
|------|------|----------|
| 标准线 | 工厂提货→义乌入库→干线运输→宁波排队→入库完成 | 大部分货物 |
| 直达线 | 工厂提货→干线运输→宁波排队→入库完成 | 急件，不经义乌仓 |
| 直送线 | 工厂提货→宁波排队→入库完成 | 工厂直接送宁波 |

**每个节点包含：**
- 状态（待处理/进行中/已完成）
- 操作员+联系电话
- 现场照片
- 时间戳
- 备注

#### 功能3：实测定价与费用计算

**仓管员实测录入：**
- 件数、重量、体积 + 现场拍照（最多6张）
- 自动回填进仓单数据作为参考
- 异常防呆：实测体积与申报差异超30%时弹窗二次确认
- 删除云端废弃照片，防止产生孤儿文件持续扣费

**费用自动计算：**

| 路线 | 计费规则 | 价格 |
|------|----------|------|
| 萌恒路线 | 体积/重量比≥4按体积计费，否则按重量 | 55元/立方 或 190元/吨 |
| 北仑路线 | 体积/重量比≥4按体积计费，否则按重量 | 85元/立方 或 220元/吨 |
| 装卸费 | 不足1立方按1立方计算 | 20元/立方 |

**附加费用（可选勾选）：**
- 代收款（按路线动态生成子项，支持编辑数量/价格）
- 分拨拼货费（≤5方100元，≤20方200元）
- 堆存费（超5天，50元/天）
- 贴唛改唛费（2元/件）
- 其他杂费（自定义名称和金额）

#### 功能4：报价与确认

**报价流程：**
```
仓管实测 → 管理员生成报价草稿 → 发送给客户 → 客户确认（现结/月结） → 自动入账
```

**客户确认页：**
- 橙色系UI，清晰展示实测照片和费用明细
- 选择现结或月结支付方式
- 二次确认弹窗，确认后不可撤销
- 事务保证：确认报价 + 创建账单 原子性操作

#### 功能5：账单管理

**客户端（工厂）：**
- 我的账单：全部/待付/已付 三标签筛选
- 月结账单合并查看
- 账单详情展示费用明细

**管理端：**
- 账单总览：统计待付/已付金额
- 标记已付（事务保证）
- 月结合并：按客户+月份批量生成月结账单
- 定时清理：每月1号凌晨自动清理已付账单

#### 功能6：双通道通知机制

**主通道：微信订阅消息**
```
【物流状态更新】26MH331011

📦 进仓编号: MSF260156150 (利盛厂)
📊 货物: 319件/5.5吨/20.3方
📍 状态: 已到达北仑仓，排队中
⏱️ 预计入库: 约15分钟
```

**兜底通道：一键复制文案**
- 当用户未绑定/未订阅时，系统自动生成文案
- 业务员复制后直接发微信/邮件

#### 功能4：权限分离体系

| 角色 | 可见范围 | 操作权限 |
|------|----------|----------|
| 管理员 | 所有货物 | 建单、更新节点、修改费用、配置费用类目、账单管理、月结合并 |
| 业务员 | 名下货物 | 查看、分享、建单（AI识别） |
| 仓库员 | 待测量队列+所有货物 | 提交实测数据、标记现结已收款、确认送达、推进物流节点 |
| 司机 | 当前运单 | 扫码查单、推进物流节点 |
| 工厂/客户 | 自己的货 | 查看、订阅通知、确认报价、查看账单 |
| 匿名（分享链接） | 单票货物 | 仅查看 |

**登录方式：**
- 业务员/仓库员：手机号登录（支持微信一键获取手机号），首次登录收集真实姓名
- 管理员：手机号+密码登录，同样收集真实姓名
- 客户：手机号绑定即可查单

#### 功能8：运单详情页编辑

**货物信息编辑（仅限仓库员/管理员）：**
- 运单号、件数、毛重、体积、唛头
- 目的港、仓库名称
- 联系人信息（姓名、电话）
- 付款方式（现结/月结）

**职责分离：**
- detail页负责信息展示和编辑
- measure页只负责实测数据录入和费用计算
- 两个页面职责分离，避免功能堆叠

#### 功能9：管理员删除运单

**软删除机制：**
- 管理员在「我的运单」和「流程」页面均可删除任意运单
- 删除不受流程状态限制（已创建/实测中/已完结均可删）
- 二次确认弹窗，防止误操作
- 删除后运单从所有列表中隐藏，但数据仍保留在数据库（`isDeleted: true`）
- 关联的查询接口（按手机号、按Token、流程列表、仓库队列等）均自动过滤已删除运单

### 3.3 数据安全设计

**核心原则：配置与数据分离**

```
货物创建时                              后续修改配置
    │                                       │
    ▼                                       ▼
┌──────────────────┐              ┌──────────────────┐
│ 路线配置快照保存   │              │  修改config文件   │
│ 到货物文档中       │              │  添加新节点       │
└──────────────────┘              └──────────────────┘
           │                                  │
           ▼                                  ▼
    历史货物保持                         新货物使用新配置
    原有路线不变                         互不影响
```

## 六、商业模式与竞争优势

### 6.1 成本结构

| 项目 | 免费额度 | 预估月费（1000票/月） |
|------|----------|----------------------|
| 云函数调用 | 100万次 | ¥0 |
| 数据库存储 | 5GB | ¥0 |
| 云存储 | 5GB | ¥0 |
| Kimi API | - | ¥20-30（约¥0.02/次） |
| **总计** | - | **¥20-30/月** |

### 6.2 竞争优势

```
传统货代                    一超物流
────────────────────────────────────────────────
电话/微信询问    →           小程序实时查看
人工录入单据     →           AI识别3秒完成
口头承诺时效     →           系统记录节点时间
月底翻找纸质单   →           电子回单一键导出
费用人工计算     →           实测自动计费
现结收款无记录   →           系统标记+账单管理
```

### 6.3 价值主张

> **"用一杯咖啡的成本，让物流服务像快递一样透明。"**
